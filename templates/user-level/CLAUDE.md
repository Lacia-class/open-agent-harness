# User-Level Open Agent Harness Routing

Use this as a safe starting point for Claude user-level `CLAUDE.md`.

## Harness Routing

When a task involves agent workflow design, artifact-backed delivery, review, SOP modeling, anti-overfitting generalization, or reusable method extraction, load the `open-agent-harness` skill or read its `core/index.md`.

## Boundaries

- Keep private profiles, credentials, account inventories, payment data, and real operational SOPs out of public repositories.
- Use adapters for runtime, validation, domain SOP resources, user preferences, terminology, and knowledge sources.
- If a private adapter is missing, continue generically and mark domain-grounded claims as unverified.
